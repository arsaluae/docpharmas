## 1. RLS coverage matrix

| Check | Result |
|---|---|
| Public tables with RLS **off** or **0 policies** | **0** ✅ |
| Public tables missing a `tenant_id` column | `pending_signups`, `role_capabilities`, `tenants`, `user_roles` — all global by design ✅ |
| Tenant-bearing tables lacking a **permissive** policy | **0** ✅ (every business table has at least one tenant-scoped permissive policy alongside the restrictive `rbac_*` ones) |
| GRANT `SELECT` to `anon` on tenant-bearing tables | **0** ✅ |

**Conclusion:** core RLS/tenant isolation is intact. No table is reachable without an authenticated session in the correct tenant.

## 2. Cross-tenant leak probes — PASS

Every public business table has both: (a) tenant filter via `get_user_tenant_id()` in a permissive policy and (b) role-capability gate via the restrictive `rbac_read/write/update` policies. Sales-agent row scoping (`is_agent_customer`) confirmed on customers + sales documents.

- `.gitignore` does **not** explicitly cover `.env.local` / `.env.*.local`. **P2** — add a pattern (Vite project; low risk but tidy).

## 7. Backup posture

- `weekly-backup` edge function exists, writes per-tenant JSON to private bucket `tenant-backups`, retains last 4. ✅
- pg_cron has **only the weekly job** (`0 0 * * 0`). **P0** — no daily, no monthly, no retention beyond 4 weekly files.
- Backup function authentication: see #3 above (**P0**).
- No `backup_runs` audit table → admins cannot see status / size / failures in the UI. **P1**.
- No restore tooling. Restore is a manual `service_role` operation. **P1** to document; **NOT** turning into a one-click button (intentional safety).
- Storage buckets:
  - `tenant-backups` — private ✅
  - `payment-screenshots` — private ✅
  - `company-assets` — **public** (logos/letterheads). OK if intentional. **P2** — confirm no PII.
  - `shared-documents` — **public** (used for share links). OK if intentional. **P2** — confirm contents are non-sensitive only.

# Severity rollup

**P0 (block launch)**
1. `weekly-backup` edge function callable by any signed-in user without owner check.
2. No daily / monthly backup cron + no retention beyond 4 files.
3. `stock_adjusted` events not in audit log.
4. `backup_created/failed/restored` not in audit log.

**P1 (before paid users)**
1. Backfill or remove 8 legacy sales lines without batch/expiry.
2. `backup_runs` table + Admin → Backups page.
3. Login / failed_login / settings_changed / CRUD audit events.
4. Function `search_path` cleanup (5 fns).
5. Enable HIBP leaked-password check.
6. Manual restore runbook (`docs/disaster-recovery.md`).

**P2 (polish)**
1. Add `.env*.local` patterns to `.gitignore`.
2. Confirm/lock down `company-assets` and `shared-documents` listing.
3. Opt into React Router v7 future flags.
4. Revoke `anon` EXECUTE on SECURITY DEFINER functions that are auth-only.

# Proposed Wave 1 scope (P0 only)

1. Migration: lock down `weekly-backup` (CRON_SECRET) + add `backup_runs` table + add daily + monthly cron jobs + retention policy.
2. Edge function update: validate `CRON_SECRET` header; write a row to `backup_runs` (status/size/error); emit `backup_created` / `backup_failed` to `audit_log`.
3. Migration + UI: `stock_adjustments` require non-empty reason at trigger level **and** a confirm-with-reason dialog client side; emit `stock_adjusted` audit event.
